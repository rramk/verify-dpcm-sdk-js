    Service

    A class for making HTTP requests to OIDC.

    new Service(auth, baseURL, context, contentTypeHeaderopt, acceptHeaderopt)

    Create a new Service object.
    Author:
    • Adam Dorogi-Kaposi
    Source:
    • services/service.js, line 10
    Parameters:
    Name Type Attributes Default Description
    auth Object The credentials to authorize requests. Only an accessToken may be used at the moment. The requests are further constrained by the entitlements on the token.
    Name Type Attributes Description
    accessToken string <optional>
    		 The access token to authorize the request.
    baseURL string The base URL for the API, normally the tenant URL.
    context Object The context to include in the request.
    Name Type Description
    subjectId string The user/subject identifier that may be a Verify user identifier.
    isExternalSubject boolean Indicates if the subject is known to Verify.
    ipAddress string The IP address of the user agent. If this library is used in a backend system, this IP should be obtained from the request headers that contain the actual user agent IP address.
    contentTypeHeader string <optional>
    		 'json' The type of content to send in the requests. Sets the Content-Type header of the requests appropriately.
    acceptHeader string <optional>
    		 'json' The type of content to receive in the response. Sets the Accept header of the requests appropriately.
